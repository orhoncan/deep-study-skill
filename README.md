# deep-study

A Claude Code skill that turns Claude into a personalized deep study companion. It creates a tailored study plan for any thinker, topic, or field based on your specific background, then guides your reading and serves as an interactive interlocutor.

## What it does

**This skill does not replace reading.** You read the actual texts. Claude plans, connects, and discusses.

The workflow has three phases:

### 1. Study Plan

Claude builds a personalized study plan based on your academic/professional background, your existing knowledge in adjacent fields, your goal, and your available time.

The plan includes a reading sequence, focus themes per text, **bridge questions** that connect the new material to what you already know, and explicit concept mappings between your field and the study topic.

> **Example:** An economist studying Goffman gets bridge questions like "How does Goffman's 'expressions given vs. given off' map onto the distinction between intentional signals and involuntary information leaks in Spence's signaling model?"

### 2. Reading Companion

While you read, Claude serves as a knowledgeable colleague — not a teacher. You can ask for:

- **Clarification** of unclear arguments
- **Connections** to adjacent literature or your own field
- **Formalization** discussions (has this idea been modeled?)
- **Critique** of weak arguments
- **Synthesis** across texts

### 3. Notes & Continuity

Key takeaways, bridges, and open questions are accumulated and saved as `.md` files. You can continue across multiple sessions — Claude picks up where you left off.

## Installation

Copy `skill.md` into your Claude Code skills directory:

```bash
mkdir -p ~/.claude/skills/deep-study
cp skill.md ~/.claude/skills/deep-study/skill.md
```

## Usage

```
/deep-study Goffman
/deep-study mimetic desire, 2 hours, no resources
/deep-study              # starts onboarding
```

Or just tell Claude you want to study something:

```
"I want to deeply study mechanism design"
```

### Commands

| Command | Action |
|---------|--------|
| `plan` | Show study plan |
| `where am I` | Progress summary |
| `notes` | Show/save notes |
| `connect {concept}` | Bridge to your domain |
| `save` | Save to file |
| `next` | Next reading + briefing |

## How personalization works

The skill checks for a `reader_persona.md` file (generated by the [build-persona](https://github.com/readwiseio/readwise-skills) skill from Readwise data). If found, it silently uses your reading history, interests, and intellectual profile to generate highly specific bridges.

Without a persona, Claude asks a few background questions and builds bridges from what you tell it. The bridges will be less specific but still personalized.

## Language

The skill auto-detects language from your first message. All output — plan, notes, commands, conversation — follows your language. Supports Turkish and English natively; works in other languages too.

## Background

This skill is based on a workflow described by [Jesús Fernández-Villaverde](https://x.com/JesusFerna7026/status/2036774895057592358) about using Claude as a personalized learning tool. The key insight: the most valuable feature of an LLM as a learning tool is personalization. A course has a single syllabus for everyone, but Claude creates one tailored for you.

# deep-study (Türkçe)

Claude'u kişiselleştirilmiş bir çalışma arkadaşına dönüştüren bir Claude Code skill'i. Arka planınıza göre herhangi bir düşünür, konu veya alan için özel bir çalışma planı oluşturur, okuma sürecinize rehberlik eder ve interaktif bir tartışma partneri olarak çalışır.

## Ne yapar

**Bu skill okumayı değiştirmez.** Asıl metinleri siz okursunuz. Claude planlar, bağlantılar kurar ve tartışır.

İş akışı üç aşamadan oluşur:

### 1. Çalışma Planı

Claude, şunlara göre kişiselleştirilmiş bir çalışma planı oluşturur: arka planınız, komşu alanlardaki bilginiz, hedefiniz ve ayırabileceğiniz zaman.

Plan; okuma sırası, metin bazında odak temaları, yeni materyali bildiğiniz alanla bağlayan **köprü soruları** ve kavram eşlemeleri içerir.

> **Örnek:** Goffman çalışan bir ekonomist, "Goffman'ın 'expressions given vs. given off' ayrımı Spence'in sinyalizasyon modelindeki kasıtlı sinyaller ve istemsiz bilgi sızıntıları ayrımıyla nasıl örtüşür?" gibi köprü soruları alır.

### 2. Okuma Eşlikçisi

Okurken Claude bilgili bir meslektaş olarak çalışır — hoca değil. Şu türlerde destek alabilirsiniz:

- Belirsiz argümanların **açıklanması**
- Komşu literatür veya kendi alanınızla **bağlantı**
- **Formalizasyon** tartışması (bu fikir modellenmiş mi?)
- Zayıf argümanların **eleştirisi**
- Metinler arası **sentez**

### 3. Notlar ve Süreklilik

Temel çıkarımlar, köprüler ve açık sorular `.md` dosyalarına kaydedilir. Birden fazla oturumda devam edebilirsiniz — Claude kaldığınız yerden devam eder.

## Kurulum

`skill.md` dosyasını Claude Code skills dizininize kopyalayın:

```bash
mkdir -p ~/.claude/skills/deep-study
cp skill.md ~/.claude/skills/deep-study/skill.md
```

## Kullanım

```
/deep-study Goffman
/deep-study mimetik arzu, 2 saat, kaynak yok
/deep-study              # tanışmayla başlar
```

Ya da Claude'a bir şey çalışmak istediğinizi söyleyin:

```
"Bir konu çalışmak istiyorum: fenomenoloji"
```

### Komutlar

| Komut | İşlem |
|-------|-------|
| `plan` | Çalışma planını göster |
| `neredeyim` | İlerleme özeti |
| `notlar` | Notları göster/kaydet |
| `bağla {kavram}` | Kavramını alanına bağla |
| `kaydet` | Dosyaya kaydet |
| `sonraki` | Sıradaki okuma + briefing |

## Kişiselleştirme nasıl çalışır

Skill, `reader_persona.md` dosyasını kontrol eder (Readwise verisinden [build-persona](https://github.com/readwiseio/readwise-skills) skill'i ile oluşturulur). Varsa, okuma geçmişinizi ve entelektüel profilinizi kullanarak çok spesifik köprüler oluşturur.

Persona yoksa Claude birkaç arka plan sorusu sorar. Köprüler daha az spesifik olur ama yine kişiselleştirilmiştir.

## Dil

Skill, ilk mesajınızdan dili otomatik algılar. Tüm çıktılar — plan, notlar, komutlar, konuşma — dilinizi takip eder.

## Arka plan
